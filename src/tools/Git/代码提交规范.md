# 代码提交规范

#### [type][scope][subject]

### Type

#### feat: 新功能
#### fix: bug修复
####
#### docs: 文档
#### style: 样式、格式
#### refactor: 重构（既不是新增功能，也不是修改bug的代码改动）
#### perf: 优化相关（提升性能、体验）
#### test: 增加测试
#### chore: 构建过程或辅助工具的变动
#### revert: 回滚到上一个版本
#### merge: 代码合并
#### sync: 同步主线或分支

### scope(可选)
说明commit影响的范围，如果多个可以使用*代替

### subject
commit目的的简短描述，最好不要超过50个字符
